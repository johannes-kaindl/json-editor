# 0.3.0 — Code Quality & Infra — Design-Spec

**Projekt:** `obsidian-json-editor`
**Datum:** 2026-05-27
**Status:** Spec, freigegeben zur Umsetzung (autonomer Run)
**Entstanden aus:** AGENTS.md Backlog §v1.3 + autonomes Roadmap-Sweep nach 0.2.0

Dieser Release fokussiert nicht auf User-Features sondern auf **Code-Hygiene und Accessibility** — drei unabhängige Sub-Bereiche, alle defaults-on.

---

## 1. Ziel & Kontext

0.2.0 lieferte sichtbare Funktionalität (Search). 0.3.0 schließt drei interne Lücken:

1. **Render-Duplizierung** — `renderObject` und `renderArray` in `src/core/render.ts` haben jeweils ~65 LOC identisches Scaffolding (Toggle, Brackets, Chip, Container, Auto-Collapse). Vor 1.0.0 (structural editing) muss das eine gemeinsame Funktion werden, sonst werden Edit-Features 2x implementiert.
2. **Accessibility-Lücke** — Der Tree ist heute mausnur. Keine `role`-Attribute, kein Tab-Stop in der Tree, kein Keyboard-Walk. Obsidian-Reviewer und User mit Screenreadern haben keine Chance.
3. **Coverage-Blindheit** — 181 Tests, aber keine Coverage-Messung. Vor Structural Editing (1.0.0) brauche ich verlässliche Coverage-Daten um zu sehen wo Lücken sind.

---

## 2. Design-Prinzipien

1. **Keine sichtbare UX-Änderung.** Visual diff = null. Wer den Plugin nur klickt, merkt 0.3.0 nicht (außer Tab-Stop und Outline).
2. **Keine neuen Settings.** Coverage und ARIA sind on-by-default.
3. **Render-Refactor mit Test-Parität.** Vor und nach dem Refactor sind alle 181 Tests grün. Plus: ein paar neue Tests die das Shared-Scaffolding direkt verifizieren.
4. **ARIA per WAI-ARIA Tree-Pattern.** Etabliertes Standard-Muster, keine Erfindung.
5. **Keyboard-Nav minimal aber komplett.** Pfeile + Enter/F2 + Home/End. Kein Cmd+G (kommt evtl. mit 0.2.1).

## 4. Sub-Bereich B — ARIA + Keyboard Navigation

### 4.1 ARIA-Rollen

| Element | ARIA-Rolle | Properties |
|---|---|---|
| `.json-tree-root` | `tree` | `aria-label="JSON content"` |
| `.json-container` (Object/Array) | `treeitem` | `aria-expanded="true|false"` (synced mit `.is-collapsed`) |
| `.json-content` (inside container) | `group` | — |
| `.json-row` mit primitive value | `treeitem` | — |
| `.json-row` mit Container als Wert | `treeitem` | (Container darin trägt `aria-expanded`) |

Die Tree-Root selber bekommt `tabindex="0"` als single Tab-Stop. Innerhalb der Tree wandert Focus via Pfeiltasten (Roving-Tabindex-Pattern).

### 4.2 Keyboard-Navigation

| Key | Action |
|---|---|
| `Tab` | Focus auf Tree-Root |
| `↓` / `↑` | Nächste / vorherige sichtbare Row (skip hidden) |
| `→` auf collapsed Container | Expandieren |
| `→` auf expanded Container | Focus erstes Child |
| `←` auf expanded Container | Collapsen |
| `←` auf collapsed Container oder Leaf | Focus Parent Row |
| `Home` | Focus erste Row |
| `End` | Focus letzte sichtbare Row |
| `Enter` / `F2` auf editable Primitive | Open Inline-Edit (gleicher Code-Pfad wie Click) |
| `Esc` während Edit | Cancel (existiert schon) |
| `Esc` außerhalb Edit | No-op (lass Obsidian's Default handhaben) |

### 4.3 Implementation

- **Roving Tabindex:** Alle Rows haben `tabindex="-1"`, die aktuell fokussierte Row hat `tabindex="0"`. Tree-Root delegiert Focus an die zuletzt aktive Row beim Tab-In.
- **Focus-Tracking:** Eine private `activeRow: HTMLElement | null` in TreeView. Initial = erste Row nach `setValue`.
- **Visible-Row-Helper:** Wenn Filter aktiv, "next visible" überspringt versteckte Rows (`.json-row:not([hidden])` reicht nicht — wir nutzen CSS `display: none` via `.json-filter-active`. Visible-check muss `offsetParent !== null` prüfen oder den Klassenset checken).
- **Container expand/collapse via keyboard:** Re-use die toggle-Click-Logik. Ich extrahiere `toggleContainer(container, force?: boolean)` als Helper, der dann sowohl vom click-Handler als auch vom Keyboard-Handler aufgerufen wird.

### 4.4 Out of Scope

- Screen-Reader-spezifische Announcements (`aria-live`) — späteres Polish.
- Multi-Select.
- Type-ahead Search (`Type to focus key starting with…`).
- Keyboard-shortcut-Discovery-Help (`?` overlay).

## 5. Sub-Bereich C — `renderObject` / `renderArray` Refactor

### 5.1 Scope

Extrahiere die geteilte Scaffolding-Logik in eine `renderContainer(kind, items, opts)` Funktion. `renderObject` und `renderArray` werden zu dünnen Wrappers, die nur die per-Item-Render-Logik (Key+Value vs Index+Value) als Callback reinreichen.

### 5.2 Neue Signature

```ts
type ContainerKind = "object" | "array";

interface ContainerItem {
  // Path-Segment this item is keyed by (string for object, number for array)
  segment: string | number;
  // Value at that position
  value: JsonValue;
}

function renderContainer(
  parent: HTMLElement,
  items: ContainerItem[],
  path: JsonPath,
  depth: number,
  opts: RenderOptions,
  kind: ContainerKind
): void;
```

Helper-Funktionen `bracketsFor(kind)`, `chipLabelFor(count, kind)`, `keyOrIndexElement(segment, kind)` werden lokal extrahiert.

### 5.3 Test-Parität

- Alle existierenden 26 `render.test.ts`-Tests müssen weiter laufen.
- Plus ~3 neue gezielt-test:
  - Empty-Object und Empty-Array gehen über `renderContainer(kind, [])` (oder einen frühen Return — wie heute).
  - Container-DOM (toggle, openBracket, chip, content, closeBracket) ist für object und array identisch in Struktur.

### 5.4 Out of Scope

- Performance-Optimierungen (virtualisierung etc.).
- Andere Render-Refactors außerhalb Object/Array.
- Aufspalten von `render.ts` in mehrere Files.

## 8. Edge Cases

| Fall | Verhalten |
|---|---|
| Tab-Stop in Tree, dann Cmd+F → Filter aktiv → Tab zurück | Activerow muss noch sichtbar sein (sonst auf erste sichtbare Row springen). |
| Empty-Object/Array | Kein treeitem, nur `.json-bracket` mit `{}`/`[]`. Kein focus möglich. OK. |
| Roving tabindex + auto-collapsed Container | Initially active row = erste sichtbare Row, nicht das collapsed inner. |
| Pfeiltasten auf collapsed Container am Ende → ↓ tut nichts (gibt keine nächste Row) | Default — kein Wrap-around. |
| Keyboard-Enter auf null-Value | No-op (null ist nicht editierbar, schon vorhanden). |
| Filter aktiv + Pfeiltasten | Walken nur visible (`.json-filter-active`-aware). |

---

## 9. Risiken

- **Roving tabindex + Inline-Edit Interaktion:** Inline-Edit ersetzt den value-Span mit Input. Wenn der Input geblurred wird, muss Focus zurück auf die Row (nicht weg vom Tree). → Im finish() callback nach render() explizit `activeRow.focus()`.
- **ARIA-aria-expanded on auto-collapsed:** Container die per `autoCollapseDepth` collapsed sind, müssen ihr `aria-expanded="false"` direkt beim Render bekommen. Toggle-Click muss `aria-expanded` synchron mit `.is-collapsed` halten.
- **Keyboard-events + global Cmd+F:** Mein Cmd+F-Command vom 0.2.0 hat `checkCallback` der nur greift wenn JsonFileView active ist. Pfeiltasten sollen nicht in Cmd+F-Shortcut beißen — sind sie nicht.
