# Obsidian JSON Editor v1.1 — Visual Polish Design

**Status:** Draft (awaiting user review)
**Date:** 2026-05-20
**Author:** Johannes Kaindl
**Predecessor:** v1.0 (commit `0d02666`, release tag `0.1.0`)

## Context

v1.0 ships a functional JSON viewer/editor for Obsidian. Manual testing surfaced the gap between "works" and "feels polished": no path orientation in deep structures, no copy convenience, no per-value introspection, abrupt collapse behavior, and a uniformly monospace look that visually flattens key/value distinction.

v1.1 closes that gap with five additive polish features, none of which alter v1.0's data model, persistence path, or core API. The previously identified Type-Icons feature was dropped during scoping — JSON type colors already serve that purpose, and icons would add visual noise that competes with the colors.

## Goals

- Ship visible polish that makes the plugin feel "complete" in the v1.0 sense, without changing user workflows.
- Keep `core/` pure and untouched in behavior — only new hook callbacks are added.
- Add no new user-facing settings; defaults must be sensible enough that nobody asks for toggles in week one.
- Maintain the v1.0 architectural boundary (core/ vs obsidian/) and add no third layer.

## Non-Goals (v1.1)

- Type-Icons (rejected during scoping — collides with type colors).
- Settings for individual polish features (default on, no opt-out in v1.1).
- Mobile-specific polish (general defaults must not actively break on touch, but no special handling).
- Structural edits, search, schema validation — those remain in v1.2/v1.3/v2.0.

## Architecture

Approach **B** from brainstorming: each polish item lives in its own small module under `src/obsidian/`. `core/render.ts` grows two new optional callbacks but stays focused.

```
src/
├── core/
│   ├── render.ts          (modified: + onPathClick, onValueHover hooks)
│   └── types.ts           (modified: + new option fields)
└── obsidian/
    ├── Breadcrumb.ts      NEW — segment-rendering component
    ├── CopyButton.ts      NEW — factory: createCopyButton(value, path)
    ├── Tooltip.ts         NEW — singleton hover-tooltip manager
    ├── TreeView.ts        (modified: wire new modules into tree)
    └── JsonFileView.ts    (modified: mount breadcrumb in chrome)

styles.css                 (modified: transitions, typography, hover, copy styles)
```

**Mount hierarchy inside `JsonFileView.contentEl`:**

```
contentEl
├── .json-mode-toggle      (unchanged)
├── .json-breadcrumb       NEW
├── .json-error-banner     (unchanged, conditional)
└── .json-editor-body      (unchanged container)
```

Tooltip lives on `document.body`, mounted once per plugin load, recycled across all hovers.

### Type additions in `core/types.ts`

```typescript
export interface RenderOptions {
  // ...existing fields
  onPathClick?: (path: JsonPath) => void;
  onValueHover?: (target: HTMLElement, path: JsonPath, value: JsonValue) => void;
}
```

Both hooks are optional. Existing callers (CodeblockProcessor, anyone else) keep working unchanged.

### Render-time changes in `core/render.ts`

In addition to firing the new hooks, `render.ts` annotates each `.json-row` with a `data-path` attribute whose value is the serialized path (using `pathToString` semantics from `CopyButton.ts`). This enables `TreeView.handleSegmentJump` to locate any row via a single `querySelector` call without recursive lookup. The attribute is added on every row regardless of whether the new hooks are wired — it's just a stable identifier.

## Per-Feature Design

### Breadcrumb

**Module:** `src/obsidian/Breadcrumb.ts`

**Public API:**
```typescript
export class Breadcrumb {
  constructor(opts: { onSegmentClick?: (subPath: JsonPath) => void });
  setPath(path: JsonPath): void;
  getElement(): HTMLElement;  // caller is responsible for mounting it where wanted
}
```

`JsonFileView` constructs the breadcrumb and uses `contentEl.insertBefore(breadcrumb.getElement(), bodyEl)` to mount it between toggle pills and body. This gives the caller full control over position without coupling the breadcrumb to a specific parent.

**DOM:**
```html
<div class="json-breadcrumb">
  <span class="bc-seg bc-seg-root">root</span>
  <span class="bc-sep">›</span>
  <span class="bc-seg">users</span>
  <span class="bc-sep">›</span>
  <span class="bc-seg">[0]</span>
  <span class="bc-sep">›</span>
  <span class="bc-seg">name</span>
</div>
```

**Behavior:**
- Empty path renders just `root` segment.
- Numeric path segments render as `[N]`; string segments render as their key.
- Each segment is clickable. Clicking fires `onSegmentClick(pathPrefix)` where `pathPrefix` includes every segment up to and including the clicked one. Clicking `root` fires with `[]`.
- Segments use Obsidian's `--text-muted` for non-last, `--text-normal` for the terminal (rightmost) segment.

**Trigger from TreeView:**
- TreeView passes `onPathClick: (path) => breadcrumb.setPath(path)` into `renderTree`.
- `renderTree` attaches a `click` listener to each `.json-row` that fires `onPathClick(path)`. Click on the value (already wired to `onValueClick` for editing) propagates the path-click first via event ordering, then triggers the inline editor.

**Segment-click jump-to-position:**
- The breadcrumb's `onSegmentClick` callback in `TreeView` finds the matching `.json-row` by data-path attribute (added in `render.ts` for this purpose), then scrolls it into view via `scrollIntoView({ block: "center", behavior: "smooth" })` and applies a brief flash class (`.json-row-flash`) for 600ms.

### CopyButton

**Module:** `src/obsidian/CopyButton.ts`

**Public API:**
```typescript
export function createCopyButton(value: JsonValue, path: JsonPath): HTMLButtonElement;
export function pathToString(path: JsonPath): string;
```

**`pathToString` rules:**
- Empty path → `"root"`.
- String segment → preceded by `.` (or no separator if first segment): `users[0].name`.
- Numeric segment → bracketed: `users[0]`.
- Keys with non-identifier characters → bracket-notation with quotes: `["weird key"][0]`.

**Behavior:**
- Click → `navigator.clipboard.writeText(JSON.stringify(value, null, 2))`.
- Alt+Click (also Option+Click on Mac) → `navigator.clipboard.writeText(pathToString(path))`.
- After successful copy, swap icon textContent from `⧉` to `✓` and add class `.copied` for 800ms, then revert.
- If clipboard API fails (rejected promise), the button stays as-is — no toast or error UI in v1.1; downstream user feedback will tell us if a Notice is needed.

**Render placement:**
- Each `.json-row` ends with this button after any comma. Default opacity 0; CSS rules show it on row hover.
- Skipped for the root-level row when path is empty AND the root is itself a primitive (rare; safe to always include the button — it always has SOME value/path to copy).

### Tooltip

**Module:** `src/obsidian/Tooltip.ts`

**Public API (singleton-style):**
```typescript
export class Tooltip {
  constructor();  // mounts to document.body
  show(target: HTMLElement, content: { typeLabel: string; pathStr: string; preview?: string }): void;
  hide(): void;
  destroy(): void;
}
```

**Behavior:**
- `show()` is called with a 500ms debounce from `TreeView.handleValueHover()`.
- On `mouseleave` of the target, `hide()` is called immediately; if `show()` was scheduled but not fired, the scheduled call is cancelled.
- Suppressed when an inline-edit input is active in the tree (TreeView tracks edit state and skips `onValueHover` calls during edits).
- DOM:
  ```html
  <div class="json-tooltip" hidden>
    <div class="tt-meta"><span class="tt-type">string</span> · <span class="tt-path">root.users[0].name</span></div>
    <div class="tt-preview">"jay"</div>
  </div>
  ```
- Position: `position: absolute`, set via JS to `target.getBoundingClientRect().left/bottom + window.scrollY`. Flip above the target if `bottom + tooltip.height > window.innerHeight`.

**Content rules:**
- Primitives: `typeLabel` is `"string"/"number"/"boolean"/"null"`; `preview` is the value as JSON-string (for strings: the actual text content without surrounding quotes, truncated to 200 chars + `…` if longer).
- Objects: `typeLabel` is `"object"`; `preview` is `"N entries"` where N is `Object.keys(value).length`.
- Arrays: `typeLabel` is `"array"`; `preview` is `"N items"`.

### Collapse Transitions

CSS-only. Update `.json-content` rules in `styles.css`:

```css
.json-content {
  max-height: 5000px;
  opacity: 1;
  overflow: hidden;
  transition: max-height 150ms ease-out, opacity 100ms ease-out;
}
.json-content.collapsed {
  max-height: 0;
  opacity: 0;
}
@media (prefers-reduced-motion: reduce) {
  .json-content {
    transition: none;
  }
}
```

`display: none` is dropped in favor of `max-height: 0 + overflow: hidden`. Existing tests that check `.json-content.collapsed` class membership still pass (only the visual mechanism changes).

**Edge case — trees deeper than ~50 levels with ~100 KB per level:** `max-height: 5000px` won't accommodate the full content during expand. Fallback behavior: the user sees the first ~5000px of expanded content; remaining content is clipped during the transition but readable once the transition completes (max-height reverts to 5000px which is below actual; we change to `max-height: unset` after transition-end).

**On transition-end:** add a `transitionend` listener on `.json-content` (only the parent, not bubbled children) that unsets `max-height` when transitioning to expanded state. Re-applies `max-height: 5000px` right before `.collapsed` is added.

### Typography & Hover

CSS-only changes in `styles.css`:

- Keys: `font-family: var(--font-interface); font-weight: 600;`
- Indent-line opacity: `var(--background-modifier-border)` (existing) → no change, already correct
- Row vertical padding: `2px 0` → `3px 6px` (added horizontal for hover-background breathing)
- New rule:
  ```css
  .json-row {
    border-radius: 3px;
    transition: background 80ms ease-out;
  }
  .json-row:hover {
    background: var(--background-modifier-hover);
  }
  ```

Tree-markers (classic style) stay monospace for column alignment.

## Data Flow

### Row click → breadcrumb update

1. User clicks anywhere on `.json-row` (key, value, comma, or whitespace).
2. `render.ts` row-listener fires `onPathClick(path)`.
3. `TreeView.handlePathClick(path)` calls `breadcrumb.setPath(path)` via the wired callback.
4. If the click target is the value span, `onValueClick` also fires (for inline edit) — both handlers run; their effects are independent.

### Value hover → tooltip

1. User's mouse enters a primitive span (`.json-string/.number/.boolean/.null`).
2. `render.ts` emits `onValueHover(target, path, value)`.
3. `TreeView.handleValueHover` checks: are we currently editing? If yes, return. Else, set a 500ms timer.
4. Timer fires: `tooltip.show(target, { typeLabel, pathStr, preview })`.
5. Mouse leaves the span → timer cleared if pending, or `tooltip.hide()` called.

### Copy button click

1. Button click → handler reads its closure: `value` and `path`.
2. Branch on `e.altKey`:
   - false → `navigator.clipboard.writeText(JSON.stringify(value, null, 2))`
   - true → `navigator.clipboard.writeText(pathToString(path))`
3. On promise resolve: add `.copied` class to button, set textContent to `✓`, schedule cleanup after 800ms.

### Breadcrumb segment click → jump

1. User clicks a non-root segment.
2. `onSegmentClick(subPath)` fires with the path up to that segment.
3. `TreeView.handleSegmentJump(subPath)` queries `[data-path="<serialized-subpath>"]` in the rendered tree.
4. If found: `el.scrollIntoView({ block: "center", behavior: "smooth" })` + add `.json-row-flash` class for 600ms.
5. If not found (path no longer valid, e.g. after structural change in v2.0): no-op.

## Error Handling & Edge Cases

| Case | Strategy |
|---|---|
| `navigator.clipboard` unavailable (older Electron) | Catch on `writeText` rejection; button stays neutral. No notice for v1.1. |
| Tooltip race during fast hover-leave | Timer cleared in `mouseleave`; tooltip never appears. |
| Tooltip overlapping inline-edit input | Suppress hovers while `inputEl.matches(":focus-within")` returns true. |
| Breadcrumb path with non-identifier keys | Render the literal key text; `pathToString` for copying uses bracket-notation. |
| Path segment click on stale path (target gone) | Silent no-op (catch on querySelector returning null). |
| Transition mid-collapse interrupted | Browser handles natively (transition reverses); no JS coordination needed. |
