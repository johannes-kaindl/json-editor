# 1.1.0 — Drag-Drop Reorder + Type-Switching — Design-Spec

**Projekt:** `obsidian-json-editor`
**Datum:** 2026-05-27
**Status:** Spec, freigegeben (autonomer Run nach 1.0.0)
**Entstanden aus:** Backlog-Item §2 in `AGENTS.md` / dekomponierte 1.0.0-Roadmap.

1.0.0 hat den Tree-Mode CRUD-fähig gemacht. 1.1.0 ist Polish: **Reihenfolge** ändern (Array-Items + Object-Keys) per Drag-Drop, und **Typ wechseln** für jeden Wert.

---

## 1. Scope

**IN:**
- `moveArrayItem(parentPath, fromIdx, toIdx)` — Array-Item innerhalb desselben Arrays verschieben
- `moveObjectKey(parentPath, key, toPos)` — Object-Key innerhalb desselben Objects neu einsortieren (Insertion-Order)
- `changeType(path, newType)` — Wert in anderen Typ konvertieren (lossy, mit sensiblen Defaults)
- UI: Drag-Handle (links der Row, hover-revealed) + TypeMenu (button "T" in RowActions)
- History-Integration via `applyMutation`

**OUT** (zukünftige Releases / nicht in 1.1.0):
- Cross-Container-Drag (z.B. Item aus Array A nach Array B verschieben)
- Touch-Gesten / Mobile-Drag
- Verschachtelte Typ-Konvertierungen mit Daten-Erhaltung (z.B. `{a:1, b:2}` → `[1,2]`); container ↔ primitive verliert immer Inhalt; container ↔ container behält nichts (nur Default leerer Container)

---

## 2. Design-Prinzipien

1. **Same-Parent-only.** Drag respektiert Hierarchie — Item bleibt im selben Container. Vereinfacht UX und schließt schwere Edge-Cases (Pfad-Validierung, Type-Mismatch) aus.
2. **Type-Switch ist destruktiv.** Konvertierung verwirft den alten Wert und erzeugt einen sensiblen Default (`""`, `0`, `false`, `null`, `{}`, `[]`). Undo macht es rückgängig — kein Confirm.
3. **Konsistenz mit 1.0.0.** Drag-Handle + Type-Button leben in/neben dem bestehenden `RowActions`-Element. Hover-Pattern, Keyboard-Erreichbarkeit, ARIA bleibt.
4. **HTML5 Drag-Drop API.** Keine externen Dependencies. `draggable=true` + `dragstart`/`dragover`/`drop`/`dragend`.

---

## 3. Core Operations (`src/core/edit.ts`)

```ts
export function moveArrayItem(value: JsonValue, parentPath: JsonPath, fromIdx: number, toIdx: number): JsonValue;
export function moveObjectKey(value: JsonValue, parentPath: JsonPath, key: string, toPos: number): JsonValue;

export type JsonType = "string" | "number" | "boolean" | "null" | "object" | "array";
export function changeType(value: JsonValue, path: JsonPath, newType: JsonType): JsonValue;
```

**Validierung / Verhalten:**

- `moveArrayItem`:
  - `fromIdx === toIdx` → No-op (returns same reference)
  - `fromIdx` out of bounds → throw `"Index out of bounds"`
  - `toIdx` clamped to `[0, length-1]` after removal
- `moveObjectKey`:
  - Key fehlt → throw `"Key not found"`
  - `toPos` clamped to `[0, length-1]`
  - Pos identisch zu aktueller → No-op
- `changeType`:
  - `newType === currentType` → No-op
  - Defaults: `string→""`, `number→0`, `boolean→false`, `null→null`, `object→{}`, `array→[]`
  - Root path (`[]`) erlaubt — gesamten Wert ersetzen
  - Container-Defaults verwerfen ihren Inhalt (siehe Scope OUT)

Alle Ops bleiben **immutable** und **pure**.

## 4. UI Surface

### 4.1 Drag-Handle

- Element: `<span class="json-drag-handle">⋮⋮</span>` als ersten Kindknoten von `.json-row`
- Zeigt nur bei Hover oder Keyboard-Fokus (CSS `opacity: 0 → 1` on `.json-row:hover`/`:focus-within`)
- `draggable="true"` direkt auf `.json-row` (Standard HTML5)
- Bei `dragstart`: Setze `dataTransfer.effectAllowed = "move"` + serialisiere Quell-Path in `dataTransfer.setData("application/json", JSON.stringify(path))`
- Bei `dragover`: Wenn Ziel-Row im selben Parent-Container — `preventDefault()` + CSS-Klasse `json-drop-target` setzen
- Bei `drop`: Path-Vergleich → wenn `parentPath` identisch → `onMoveItem`/`onMoveKey` callback
- Bei `dragend`: Alle `json-drop-target`/`json-dragging` Klassen entfernen

**Insertion-Position:** Drop-Position relativ zur Ziel-Row — wenn Mouse in oberer Hälfte: vor Ziel; untere Hälfte: nach Ziel. Visuelle Indikator-Linie (`::before`/`::after` mit `border-top`/`border-bottom`).

### 4.2 Type-Menu

- Neuer Button `T` in `RowActions` (links von ✎, wenn vorhanden — sonst links von ✕)
- Click öffnet inline-popover unter der Row mit 6 Optionen:
  - `String`, `Number`, `Boolean`, `Null`, `Object`, `Array`
- Aktueller Typ ist disabled / als active markiert
- Klick auf Option → `onChangeType(path, newType)` + popover schließt
- ESC oder click-outside schließt popover
- Implementierung: neue Component `TypeMenu.ts`

## 5. TreeView-Erweiterungen

Neue Opts auf `TreeViewOptions`:

```ts
onMoveItem?: (parentPath: JsonPath, fromIdx: number, toIdx: number) => void;
onMoveKey?: (parentPath: JsonPath, key: string, toPos: number) => void;
onChangeType?: (path: JsonPath, newType: JsonType) => void;
```

Wiring:
- `attachStructuralActions` erweitert um Drag-Handle-Anbringung an jeder Row
- `RowActions` bekommt optionalen `onChangeType: () => void` callback → TreeView öffnet `TypeMenu`

---

## 6. JsonFileView-Handler

Drei neue handler analog zu `handleDelete`:

```ts
private handleMoveItem(parentPath: JsonPath, fromIdx: number, toIdx: number)
private handleMoveKey(parentPath: JsonPath, key: string, toPos: number)
private handleChangeType(path: JsonPath, newType: JsonType)
```

Alle gehen durch `applyMutation` mit beschreibendem Label (`"Reorder item"`, `"Reorder key"`, `"Change type to <type>"`).

## 8. Out-of-Scope-Risiken (dokumentiert)

- **happy-dom + HTML5-Drag**: happy-dom unterstützt `DataTransfer` rudimentär; Tests verwenden direkte `dispatchEvent`-mocks statt echter Drag-Operationen.
- **Drop-Position Heuristik (top/bottom half)**: in Unit-Test schwer zu validieren; deshalb separate logische `computeInsertionIndex(fromIdx, toIdx, dropPosition: "before"|"after")` Pure-Function — die wird unit-getestet, der Mouse-Y-Vergleich nicht.
