# 0.3.0 — Code Quality & Infra — Implementation Plan

**Spec:** `docs/superpowers/specs/2026-05-27-0.3.0-quality-infra-design.md`
**Branch:** `feat/0.3.0-quality-infra` (already created)
**Goal:** Coverage tooling + ARIA tree + keyboard navigation + render-refactor.

**Execution order chosen for least churn:**
1. Coverage tooling (independent, smallest).
2. Render-refactor (changes `render.ts` structure; ARIA additions land cleanly afterwards).
3. ARIA roles (built on refactored renderContainer).
4. Keyboard nav (uses ARIA roles to identify treeitem rows).
5. Release artifacts + merge + tag.

## Phase 3 — ARIA roles

- [ ] In `renderTree`: add `role="tree"` + `aria-label="JSON content"` to `.json-tree-root`
- [ ] In `renderContainer`: add `role="treeitem"` + `aria-expanded` (true/false based on initial collapse state) to `.json-container`
- [ ] In `renderContainer`: add `role="group"` to the inner `.json-content` div
- [ ] In each per-item render: add `role="treeitem"` to the `.json-row`
- [ ] Toggle handler updates `aria-expanded` on the container alongside the existing class toggle
- [ ] Add `tests/core/render.aria.test.ts` with ~6 tests
- [ ] Commit: `feat(core): WAI-ARIA tree roles + aria-expanded sync`

## Phase 4 — Keyboard navigation

- [ ] In `TreeView`: track `private activeRow: HTMLElement | null` (default = first visible row after render)
- [ ] Roving tabindex: all `.json-row` get `tabindex="-1"`, activeRow gets `tabindex="0"`. Tree root focuses delegate to activeRow.
- [ ] Add keydown handler on tree root that handles ↓ ↑ → ← Home End Enter F2
- [ ] Helper `visibleRows(): HTMLElement[]` walks tree DOM, filters via `.json-filter-active`-aware visibility check (offsetParent !== null is the simplest)
- [ ] Helper `toggleContainer(container, force?)` reused by both click and keyboard handlers
- [ ] After inline-edit finish() and re-render, restore focus to the same path (`activeRow` refreshed via `data-path` lookup)
- [ ] Add `tests/obsidian/TreeView.keyboard.test.ts` with ~12 tests
- [ ] Commit: `feat(obsidian): keyboard navigation through the tree (arrows + Home/End + Enter/F2)`
